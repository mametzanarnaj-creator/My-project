1. Жобаны жоспарлау (Management)
1.1 Әдістемені таңдау

Жобаны әзірлеуде Scrum әдістемесі қолданылады.

Scrum элементтері:

Өнім бэклогы (Jira немесе GitHub Projects)

2 апталық спринттер

Күнделікті жиналыс (Daily Sync – 15 минут)

Спринт соңындағы шолу және ретроспектива

1.2 Рольдік модель

Project Manager / Scrum Master – жобаны басқару, мерзімді бақылау

System Analyst – талаптарды анықтау, жүйе сипаттамасын жасау

Frontend / Backend Developer – бағдарламалық кодты жазу

DevOps Engineer – CI/CD және инфрақұрылымды баптау

1.3 Іске асыру кезеңдері (Roadmap)

1-ай: Деректер базасының архитектурасын жобалау және технологиялық стек таңдау

2-ай: MVP (ең төменгі жұмыс істейтін өнім) әзірлеу

3-ай: Тестілеу, қателерді түзету және 1.0 нұсқасын шығару

2. Бағдарламалық жасақтама инфрақұрылымы
2.1 Технологиялық стек

Нұсқаларды бақылау жүйесі (VCS): GitHub

Контейнерлеу: Docker

Оркестрация: Kubernetes

2.2 CI/CD конвейері

Кодты репозиторийге жіберу (Commit)

Автоматты тестілеу

Қауіпсіздік тексерісі

Docker бейнесін жасау

Staging ортасына орналастыру (Deploy)

2.3 Infrastructure as Code (IaC)

Инфрақұрылым Terraform арқылы сипатталады.
Бұл тәсіл серверлер мен сервистерді автоматты түрде іске қосуға мүмкіндік береді.

3. Практикалық бөлім (Чек-лист)
3.1 Архитектуралық схема

Frontend ↔ API ↔ Database өзара байланысы

3.2 Орталар (Environments)

Dev – әзірлеу ортасы

Stage – тестілеу ортасы

Prod – негізгі жұмыс ортасы

3.3 Мониторинг

Grafana – жүйе жүктемесін бақылау

Sentry – қателерді бақылау

3.4 Қауіпсіздік

Құпия деректер GitHub Secrets немесе Vault арқылы сақталады
